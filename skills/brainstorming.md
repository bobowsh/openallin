# Brainstorming — 头脑风暴与需求探索

> 在任何创造性工作之前使用此技能 — 通过多轮对话探索用户意图、需求和设计，然后再实现。

## 何时使用

- 用户提出新功能需求
- 需要理解模糊的需求
- 在编写代码前需要澄清意图
- 需要探索多种实现方案

## 核心设计：多轮迭代

```
┌─────────────────────────────────────────────────────────────┐
│  头脑风暴是多轮迭代过程，每轮包含：                         │
│                                                             │
│  1. AI 提出问题 → 2. 用户回答 → 3. 总结确认 → 4. 下一轮  │
│                                                             │
│  最多 5 轮，或直到用户确认"够了"                           │
└─────────────────────────────────────────────────────────────┘
```

## 每轮工作流

### 轮次结构

```
第 N 轮讨论
━━━━━━━━━━━━
1. 主题（这轮要讨论什么）
2. 讨论内容（Q&A）
3. 轮次总结（给用户确认）
4. 用户决策（继续/结束/修改）
```

### 第 1 轮：理解核心意图
- 用户真正想要解决什么问题？
- 这个需求的业务价值是什么？
- 期望的结果是什么样的？

### 第 2 轮：探索边界条件
- 输入和输出是什么？
- 有哪些边界情况？
- 什么情况算成功？什么算失败？

### 第 3 轮：环境与约束
- 与现有系统的交互点在哪里？
- 有什么技术限制？
- 性能、安全、兼容性要求？

### 第 4 轮：方案设计
- 提出 2-3 种实现方案
- 分析各方案的优缺点
- 给出推荐方案

### 第 5 轮：确认与收尾
- 明确范围（做什么 / 不做什么）
- 确认技术选型
- 制定验收标准

## 轮次总结格式

每轮结束时，输出：

```markdown
## 第 N 轮总结

### 已确认
- ...

### 待确认
- ...

### 用户决策
[ ] 继续下一轮
[ ] 结束讨论，进入实现
[ ] 需要修改前面的内容
```

## 状态保存

每轮结束后，将当前状态保存到 `workspace/brainstorm-<date>.md`：

```markdown
# 头脑风暴记录

> 日期: 2026-04-09
> 主题: <用户需求>

## 轮次历史

### 第 1 轮：...
### 第 2 轮：...
...

## 当前状态
- 轮次: N
- 已确认意图: ...
- 待解决问题: ...

## 最终输出
（讨论结束后填写）
```

## 终止条件

满足以下任一条件，讨论结束：

1. 用户明确说"够了"或"可以了"
2. 完成 5 轮讨论
3. 用户说"进入实现"
4. 超过 30 分钟（建议进入下一阶段）

## 最终输出物

讨论结束后，输出完整记录：

1. **需求澄清记录** — 用户的核心诉求
2. **方案对比分析** — 各方案优缺点
3. **推荐方案及理由** — 为什么选这个
4. **验收标准** — 怎么算做完了
5. **边界情况清单** — 需要处理的异常

## 注意事项

- 不要急于实现，先理解
- 多问"为什么"，少问"怎么做"
- 保持方案简洁，避免过度设计
- 每轮都要给用户决策权，不是单方面推进
- 如果用户回答模糊，主动追问具体细节
